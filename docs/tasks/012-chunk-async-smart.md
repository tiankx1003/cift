## 目标

### 12.1 分段异步任务化

**目标：** 点击分段后立即返回，后台异步执行分段，前端通过轮询展示进度。

**当前问题：**
- 点击分段后页面一直转圈等待，大文件分段耗时较长时体验差
- 无法同时对多个文档发起分段

**实现要求：**

#### Python 服务
- 分段接口 `POST /internal/documents/{doc_id}/chunk` 改为异步：
  - 立即返回 `task_id`，后台执行分段任务
  - 新增分段任务状态表或复用现有机制，记录：task_id、doc_id、status（pending/processing/completed/failed）、progress（0-100）、total_chunks、current_chunk、error_message
  - 分段过程中按 chunk 粒度更新进度

#### Node 服务
- 代理 Python 异步分段接口，返回 task_id
- 新增接口 `GET /api/documents/{doc_id}/chunk-progress`，代理查询分段进度

#### 前端
- 点击分段后不再 loading 阻塞，而是下发任务后 toast 提示「分段任务已提交」
- 文档列表新增「进度」列，显示分段进度百分比（如 45%）
- 进行中的进度自动轮询刷新（每 2 秒）
- 进度 100% 后显示「已完成」并刷新文档列表

### 12.2 分隔符对齐分段

**目标：** 分段时在接近目标长度的位置寻找分隔符作为实际切分点，避免在句子/段落中间截断。

**当前问题：**
- 分段完全按固定长度切分，可能在句子中间断开，导致语义残缺

**算法逻辑：**

当文本累计长度达到 `chunk_size` 时：
1. 在当前位置**向前**搜索最近的分隔符
2. 同时在当前位置**向后**搜索最近的分隔符
3. 取距离 `chunk_size` 位置更近的那个分隔符作为实际切分点
4. 如果向前搜索到 0 个字符（即当前位置恰好是分隔符），则优先使用
5. 如果向前找不到分隔符，则取向后最近的
6. 实际分段长度**不应超过 chunk_size 的两倍**，超过则强制在两倍长度处截断
7. 分段之间保留 `chunk_overlap` 的重叠区域

**预设分隔符（按优先级从高到低）：**
- `\n\n`（段落分隔）
- `\n`（换行）
- `。`（中文句号）
- `？`（中文问号）
- `！`（中文感叹号）
- `；`（中文分号）
- `.`（英文句号）
- `?`（英文问号）
- `!`（英文感叹号）
- ` `（空格，最低优先级兜底）

**实现要求：**

#### Python 服务
- 改造 `make_chunks` 函数（或分段服务模块），实现上述分隔符对齐算法
- 分隔符列表可通过分段配置覆盖，默认使用上述预设列表
- 确保算法在边界情况下稳定（文本短于 chunk_size、无任何分隔符、连续分隔符等）

#### 涉及层
