# TASK 011: 体验细化

## 目标

对知识库的文件上传、分段、展示等核心交互进行体验优化，提升日常使用的流畅度和效率。

## 背景

Phase 2 已完成知识图谱、模型管理等核心功能，但部分基础交互仍较粗糙：
- 上传只能单选文件
- 批量操作（分段、删除）缺失
- 文档列表缺少分段策略信息，操作项有冗余
- 已分段文档无法重新分段

## 子任务

### 11.1 多文件上传 ✅

**已完成。** Upload 组件添加 `multiple` 属性，支持一次选择多个文件。

---

### 11.2 文档列表分段策略信息列 ✅

**已完成。** Document 模型新增 chunk_size/chunk_overlap/separators 列，文档列表 Table 新增「分段策略」列。

**目标：** 文档列表新增分段策略相关列，让用户直观了解每个文档的分段配置。

**实现要求：**

#### 前端
- 文档 Table 新增列，展示分段策略信息：
  - 块大小（chunk_size）
  - 重叠大小（chunk_overlap）
  - 分隔符（separators）
- 未分段的文档显示「—」
- 数据来源：关联 chunk_configs 表，取该文档最近一次使用的分段配置

#### 涉及层
- 前端 + Node（可能需调整文档列表接口返回分段配置信息）

#### 复杂度
低

---

### 11.3 操作列精简 ✅

**已完成。** 移除操作列中的「查看分块」按钮，点击文档文件名即可跳转分块预览。

---

### 11.4 已分段文档支持重新分段 ✅

**已完成。** 所有文档均显示「分段」按钮，已分段文档点击时弹出确认提示。后端已支持重新分段（先删旧向量再重新分段）。

### 11.5 批量分段 ✅

---

### 11.6 文档批量删除 ✅

**已完成。** 复用行选择机制，「批量删除」按钮带 Popconfirm 确认。

---

### 11.7 模型管理页面合并 + 默认 Embedding 模型展示 ✅

**已完成。** Manage.tsx 改为三个 Card 垂直排列。Embedding Card 顶部展示 .env 配置的默认模型信息。Python 新增 `/internal/models/default-embedding` 接口。

## 执行顺序

建议按 11.2 → 11.3 → 11.4 → 11.5 → 11.6 → 11.7 顺序执行。11.2 和 11.3 纯前端改动可快速完成；11.4 涉及后端调整；11.5 和 11.6 依赖行选择机制；11.7 页面重构放最后。
