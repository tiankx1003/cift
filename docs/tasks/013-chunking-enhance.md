# TASK 013: 分段能力强化

## 目标

强化分段能力，支持按文档标题/章节结构智能切分，可选支持 LLM 智能分段和分段质量评估。

## 子任务

### 13.1 按标题/章节分段（重点）

**目标：** 识别文档中的标题层级结构（Markdown 标题、PDF 章节标题），按章节作为分段边界，保持语义完整性。

**当前问题：**
- 现有分段仅按固定长度 + 分隔符对齐，无法识别文档结构
- 一节的内容可能被拆到多个 chunk 中，或不同节的混合在一起

**实现要求：**

#### Python 服务
- 新增分段策略类型 `structural`（结构化分段），与现有 `fixed`（固定长度）并存
- 实现文档结构识别：
  - **Markdown**：解析 `#`、`##`、`###` 等标题语法，构建标题层级树
  - **TXT**：通过正则匹配常见标题模式（如「第X章」「X.Y」「一、」「（一）」等中文章节格式）
  - **PDF/DOCX**：通过字体大小、加粗等样式信息识别标题（PDF 用 PyMuPDF 的 font 信息，DOCX 用 python-docx 的 style 信息）
- 按标题层级切分规则：
  - 以指定的标题级别作为分段边界（如选择「H2」，则每个 `##` 及其下属内容为一个 chunk）
  - 如果单个章节内容超过 chunk_size，则在章节内部按分隔符对齐算法二次切分
  - 每个 chunk 保留所属章节的标题作为前缀上下文（如 `## 第二章 xxx\n内容...`）
- 分段配置新增字段：`strategy`（fixed / structural）、`heading_level`（用于 structural 策略的切分层级）

#### 前端
- 分段策略配置新增「分段策略」选择（固定长度 / 结构化分段）
- 选择「结构化分段」时，显示「切分层级」选项（H1/H2/H3/H4 或自动检测）

#### 涉及层
- 前端 + Python（核心）+ Node

#### 复杂度
高

### 13.2 LLM 智能分段（可选）

**目标：** 调用 LLM 对文档内容按语义切分，适用于结构不明显但语义边界清晰的文档。

**实现要求：**

#### Python 服务
- 新增分段策略类型 `llm`
- 将文档文本按固定长度预切分，然后调用 LLM 判断每个切分点是否为语义边界
- LLM prompt 设计：输入相邻两段文本，输出是否应该在此处切分（是/否）
- 支持配置使用的 LLM 模型（复用模型管理中的 LLM 配置）
- 降级策略：LLM 调用失败时自动退化为分隔符对齐分段

#### 前端
- 分段策略新增「智能分段（LLM）」选项
- 选择时需指定 LLM 模型

#### 涉及层
- 前端 + Python

#### 复杂度
中

### 13.3 分段质量评估（可选）

**目标：** 分段完成后自动检测每个 chunk 的质量，标记可能存在语义残缺的分块。

**实现要求：**

#### Python 服务
- 分段完成后，对每个 chunk 计算质量指标：
  - 首尾是否为完整句子（以句号/问号/感叹号结尾）
  - chunk 长度是否在合理范围内（chunk_size × 0.3 ~ chunk_size × 2）
  - 是否以标题/列表开头（结构完整性）
- 为每个 chunk 添加 `quality_score` 字段（0-100）
- 低质量 chunk（score < 60）在前端标黄提示

#### 涉及层
- Python + 前端

#### 复杂度
低
