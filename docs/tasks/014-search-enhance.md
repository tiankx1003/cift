# TASK 014: 搜索召回优化

## 目标

优化搜索召回能力，支持参数调节、权重配置、结果展示升级，提升检索体验和准确性。

## 子任务

### 14.1 搜索参数可调

**目标：** 搜索时支持用户调节召回参数，包括 top-k、相似度阈值、向量相似度权重、混合相似度阈值。

**实现要求：**

#### 前端
- 搜索框下方或侧边新增搜索参数面板（可折叠）：
  - **返回数量（top-k）**：滑块或输入框，范围 1-50，默认 10
  - **最低相似度阈值**：滑块，范围 0.0-1.0，默认 0.3，低于此值的结果不返回
  - **向量相似度权重**：当开启混合检索时，调节向量得分与关键词得分的权重比例（0.0-1.0），默认 0.7
  - **混合相似度阈值**：混合检索时，最终加权得分低于此值的结果不返回
- 参数面板默认折叠，展开后实时显示当前参数值

#### Node 服务
- 搜索代理接口透传搜索参数

#### Python 服务
- 搜索接口接收并应用参数：
  - `top_k`：限制返回数量
  - `similarity_threshold`：过滤低相似度结果
  - `vector_weight`：混合检索时的向量权重
  - `hybrid_threshold`：混合检索时的最终阈值

#### 涉及层
- 前端 + Node + Python

**目标：** 优化搜索结果页面的信息展示，支持分屏对照查看分段内容与原文。

**实现要求：**

#### 前端
- 搜索结果每条展示：
  - 来源文档名称（可点击跳转文档详情）

  - 相似度评分（百分比 + 进度条）

- 新增分屏预览功能：
  - 点击某条搜索结果，右侧/下方展开原文对照面板

  - 复用 ChunkPreview 的原文高亮逻辑
- 搜索结果支持按相似度排序（默认）/ 按文档分组
